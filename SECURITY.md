# Security Policy

## Supported versions

Security fixes are applied to the latest code on the default branch and deployed to [csp-playground.dev](https://csp-playground.dev).

| Version | Supported |
| ------- | --------- |
| Latest  | Yes       |

## Reporting a vulnerability

**Please do not open a public GitHub issue for security vulnerabilities.**

Report security issues privately using one of these channels:

1. **[GitHub Security Advisories](https://github.com/Nooshu/CSP-Playground/security/advisories/new)** (preferred)
2. Contact the maintainer via [nooshu.com](https://nooshu.com)

Include as much detail as you can:

- Description of the issue and potential impact
- Steps to reproduce
- Affected URLs, endpoints, or components (for example `/api/csp-lookup`)
- Proof-of-concept code or screenshots, if available

We aim to acknowledge reports within **5 business days** and will keep you updated on remediation progress.

## Scope

In scope:

- The CSP Builder web application and static assets
- The `/api/csp-lookup` Pages Function and shared server lookup code
- Supply-chain or build/deploy configuration that affects production

Out of scope:

- Vulnerabilities in third-party sites you import CSP from via URL lookup
- Issues that require compromising a maintainer's personal accounts or infrastructure outside this repository
- General CSP misconfiguration on sites that use policies generated by this tool

## Safe harbour

We appreciate responsible disclosure. If you follow this policy and give us reasonable time to investigate and fix an issue before public disclosure, we will not pursue legal action related to your research.
